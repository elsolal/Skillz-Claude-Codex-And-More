# Protocoles de navigation

## Site public

1. Ouvrir la homepage et noter title, URL finale, langue, cookies/consent.
2. Capturer snapshot + screenshot above-the-fold.
3. Relever navigation principale, footer, CTA, pages produits/services/pricing/about/contact/blog/docs.
4. Visiter 3-5 pages representant l'offre et le contenu.
5. Verifier console, erreurs reseau, liens cassants visibles et responsive mobile si pertinent.
6. Restituer pages visitees + informations confirmees.

## App connectee

1. Demander ou utiliser un compte test; ne jamais inventer de credentials.
2. Sauvegarder l'etat auth hors repo si necessaire.
3. Cartographier les routes accessibles, menus, empty states, flows critiques.
4. Eviter creation/suppression/paiement sans accord explicite.
5. Apres chaque action: snapshot, console, reseau, changement d'etat.

## Benchmark concurrentiel

1. Exiger URL concurrent ou source fournie.
2. Analyser la homepage et 2-4 pages clefs.
3. Extraire positionnement, offres, pricing visible, preuves sociales, CTA, angles SEO/GEO.
4. Marquer toute information non visible comme `Non vérifié`.
5. Comparer uniquement les elements observes.

## Tunnel ou flow precis

1. Definir point de depart et condition de fin.
2. Capturer l'etat initial.
3. Avancer etape par etape, sans sauter les validations visibles.
4. Noter blockers, microcopy, erreurs, latence visible, changements d'URL et requetes importantes.
5. Produire une reproduction concise.
